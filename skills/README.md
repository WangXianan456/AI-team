# Skills

Skills are reusable behavior packs attached to roles.

## Suggested Mapping

- `pm`: requirement-clarity, prioritization, acceptance-design
- `dev`: code-planning, refactor-safety, test-first-checks
- `dev`: security-hardening checks
- `qa`: test-design, regression-focus, defect-triage
- `ops`: release-safety, rollback-drill, observability-basics
- `dev/qa`: framework depth packs (backend/frontend) + pitfalls checklist

## Contract

Each skill should define:

1. Trigger condition
2. Input schema
3. Output schema
4. Guardrails
5. Failure fallback
